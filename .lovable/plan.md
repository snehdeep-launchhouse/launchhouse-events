### Quote Number Generation

A sequential 4-digit number (0001, 0002 … 9999) auto-generated by the database using a Postgres sequence. This is stored as `quote_number` (integer) on the `quote_requests` table and displayed as a zero-padded 4-digit string on the thank-you screen and in notification emails.

- Implemented as a database sequence (`quote_number_seq`) starting at 1, with `nextval()` called on insert via a `DEFAULT` column value.
- Purely backend — users never see an editable field for it.

### Edge Function — `send-quote-request`

New function at `supabase/functions/send-quote-request/index.ts`, completely separate from `send-build-request`. It:

1. Inserts the row into `quote_requests` (getting back the auto-generated `quote_number`)
2. Sends an **internal notification** to `sam@launchhouse.events` and `snehdeep@launchhouse.events` with a neat HTML table of all fields, prominently showing the Quote # in the subject line: `New Quote Request #0042 – [Full Name]`
3. Sends a **confirmation email** to the submitter: `"We've received your quote request – Quote #0042"`
4. Uses the existing `RESEND_API_KEY` secret — no new secrets needed
5. Same rate-limit handling (600ms sleep between sends) as the existing function
